HTML tags

HTML indicates structure, and only incidentally appearance. Much seriously noxious HTML comes from trying to get the HTML to do appearance, when CSS is really a far better tool for that.

Structural elements

Structural elements are part of the structure of the document, without necessarily having a whole lot to do with its appearance. So <div> marks off a division within the document, but says nothing whatever about what kind of division. Similarly, <span> marks off a span of characters, but tells nothing about why those particular characters are special. The most common use for both <div> and <span> is to hang attributes on (often style or class attributes, which we'll see in more detail later).

Headers

Header tags are also structural, but do tell something about what's special—that text is a header of the given level.

Links

Links, made with the <a> tag, make the World Wide Web actually webby, because pages link to other pages. The href attribute tells where the link goes.

A link can point within a page to any tag that has an id attribute.

Text tags

Relatively common

<p> tags enclose paragraphs. <em> is emphasized text, usually rendered in italics (but this is not guaranteed!). <strong> encloses strongly emphasized text, usually rendered in bold (again, this is not guaranteed!).

Less common

<sub> and <sup> do what they say. <br /> forces
